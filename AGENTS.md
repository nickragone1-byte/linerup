# Linerup — Agent Rules

## CRITICAL: What you must NEVER touch
- `v10-mlb-playoff.R` — the MLB prediction model. Never edit this file.
- `nba_v7_predictions.R` — the NBA prediction model. Never edit this file.
- `lib/tier.ts` — tier logic mirrors the R model output. Only edit if explicitly asked.
- `public/data/mlb/predictions.json` — generated by R script, never hand-edit.
- `public/data/nba/predictions.json` — generated by R script, never hand-edit.
- Crontab — never modify scheduled jobs.

## What you CAN change
- UI components in `app/components/`
- Page layouts in `app/mlb/page.tsx` and `app/nba/page.tsx`
- Styling, labels, copy, section names
- `lib/narrative.ts` — the text descriptions shown on cards
- `lib/display-tier.ts` — how internal tiers map to display labels

## Rules
1. Never run `crontab -e` or modify any cron jobs.
2. Never edit R scripts (.R files) for any reason.
3. Always run `npm run build` in your HEAD before pushing — if build fails, fix it before committing.
4. EV (expected value per $100) is the primary betting metric shown to users. Edge % is secondary.
5. The three user-facing sections are: Model Favorites, Leans, Cautious Plays.
6. When in doubt, ask rather than guess.
